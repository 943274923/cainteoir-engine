Audio Output

Most text-to-speech programs have limited audio output capabilities. They usually either output to an audio device through a single API such as OSS, ALSA, PortAudio or PulseAudio or to a file in a specific raw format such as wav.

By having the text-to-speech engine write to an audio interface, it is possible to support a larger number of audio APIs and file formats.

The audio API should support the following capabilities:

  1. Creating an audio connection with a specific number of channels, frequency and format;
  2. Closing a connection;
  3. Writing raw audio data to the connection;
  4. Splitting the connection — this is to support recoding audio over multiple files;
  5. Writing meta-data to the connection — this is for files that support meta-data like ogg and flac.

At a minimum, the following audio device APIs should be supported:

  1. OSS — this is to support environments that have either OSSv3 or OSSv4 such as BSD platforms;
  2. ALSA — this is to support environments that have an ALSA device such as Linux platforms;
  3. PulseAudio — this is to support environments that have a PulseAudio device such as modern Ubuntu platforms.

and the following file formats should be supported:

  1. WAV — writes the data to the Windows Wave audio format;
  2. OGG+Vorbis — writes the data to the vorbis audio format in an ogg container with meta-data support;
  3. FLAC — writes the data to the FLAC audio format with meta-data support.
