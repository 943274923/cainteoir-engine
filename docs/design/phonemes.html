Phonemes

Each text-to-speech program has its own way of representing phonetic information all of which ultimately rely on the International Phonetic Alphabet (IPA). There are two classes of phonetic representation:

  1. language-specific — these schemes are based on the subset of phonemes that constitute the language the voice supports which for English is around 40;
  2. IPA — these schemes are independent of the language used and are a way to represent the International Phonetic Alphabet;

The International Phonetic Alphabet is a way of transcribing the audio characteristics of any given phoneme as pronounced through the vocal tract. Any other phoneme set can be mapped to the same characteristics as they are all describing how the phoneme sounds, but not all phoneme sets will map to all pronouncable phonemes.

This observation alows the internal representation of the phonemes to be described through the phonetic characteristics independent of any given transcription scheme. A transcription scheme then defines how to map between this internal representation and the ASCII or Unicode characters forming the transcription.

The following transcription schemes are IPA-derived:

  1. International Phonetic Alphabet — the canonical Unicode transcription;
  2. Kirshenbaum — an ASCII representation;
  3. X-SAMPA — an alternate ASCII representation based on the different SAMPA language schemes;
  4. Z-SAMPA — an extension to X-SAMPA that forms a complete IPA representation scheme;
  5. WorldBet — an alternate ASCII representation;
  6. CXS — an extension to the X-SAMPA representation;
  7. TIPA — an ASCII representation used in TeX documents.

The following transcription schemes are language-specific, but use a notation similar to the IPA-derived ASCII representations:

  1. SAMPA — language specific transcriptions for many different languages;

The following transcription schemes are language-specific, but use spaces between each phoneme making it hard to identify word boundaries. They also don't provide any standard way to annotate this or other prosodic information apart from stress. As such, these are not likely to be supported but are listed here for reference:

  1. Arpabet — a US English transcription scheme used by the CMU pronunciation dictionary;
  2. Cepstral — Cepstral Text-to-Speech uses a variation of arpabet: US English, UK English, Italian, Canadian French, German, Americas Spanish.
